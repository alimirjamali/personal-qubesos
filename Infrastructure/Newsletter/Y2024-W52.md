### Details
In addition to the usual minor fixes and patches:

- **core-qubesdb** [v4.2.7](https://github.com/QubesOS/qubes-core-qubesdb/compare/v4.2.6...v4.2.7) (r4.2 & r4.3)
. Bumping supported Python version for Archlinux
. One part of readjusting SELinux labels for Fedora 41

- **core-admin-linux** v4.2.31 & [v4.3.6](https://github.com/QubesOS/qubes-core-admin-linux/compare/v4.3.5...v4.3.6) (r4.2 & r4.3)
. A Major part of SELinux relabling for the existing templates (via updater) to make memory ballooning work again. This one is education and one of the example cases where updating via the distro's native updating tool (dnf in this case) is not enough and user has to use the Qubes Update GUI or `qubes-vm-update` to make it work.

- **linux-utils** v4.2.19 & [v4.3.7](https://github.com/QubesOS/qubes-linux-utils/compare/v4.3.6...v4.3.7) (r4.2 & r4.3)
Another part of SELinux label fix for Fedora 41

- **gui-agent-linux** v4.2.20, [v4.3.3 & v4.3.4](https://github.com/QubesOS/qubes-gui-agent-linux/compare/v4.3.2...v4.3.4) (r4.2 & r4.3)
. Assuring GUI render is done via software for GTK.
. sys-net & sys-usb Application icons will not be sent to GUIVM (dom0) if either `minimal-netvm` or `minimal-usbvm` qube service is enabled. If user opens an application from those VMs, their icons will be the default qube icon. This is done to save small amount of RAM.
. Bumping supported Pulseaudio version to allow Archlinux to update.
. Replacing pulseaudio with pipewire on Archlinux template.

- **core-agent-linux** [v4.2.40](https://github.com/QubesOS/qubes-core-agent-linux/compare/v4.2.39...v4.2.40) & [v4.3.14](https://github.com/QubesOS/qubes-core-agent-linux/compare/v4.3.13...v4.3.14) (r4.2)
. Some parts of supporting and promoting in-vm Kernel support for r4.2 (this has been tested on r4.3).
. Bumping supported Python version to allow Archlinux to update
. The main part of allowing icon sender to be disabled on sys-usb and sys-net
