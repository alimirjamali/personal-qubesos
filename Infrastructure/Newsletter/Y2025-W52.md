##### ZRAM Support

This is a small last minute gift from some of the community and core team members. For people who are low on RAM and can not afford more memory; or their systems does not support more memory physically. ZRAM could provide dynamically resized compressed virtual memory which itself resides in RAM. Since many programs do not use RAM efficiently and/or store a lot of clear text in RAM, it could achieve 1:2 or even 1:3 memory compression (according to [Archlinux WIKI](https://wiki.archlinux.org/title/Zram)).

In the below screenshot, I have browsers open in two qubes which are under memory stress. ZRAM has achieved 1:2.1 and 1:2.4 compression respectively. It is quite safe as it resides in qube's initial memory space and does not interfere with qubes dynamic memory ballooning.

![Y2025-W52-zram|690x387](upload://3ZH6E3irHAf6hnmZWcEflkjAACj.png)

ZRAM has been used in vanilla Fedora [since 2020](https://fedoraproject.org/wiki/Changes/SwapOnZRAM) (Fedora 33). For Qubes OS Fedora templates, users cold install `zram-generator-defaults` in Fedora templates/standalones to enable it. Debian [Wiki](https://wiki.debian.org/ZRam) has some instructions on how to enable ZRAM.
