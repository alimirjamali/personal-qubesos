* **app-linux-input-proxy** [v1.0.44](https://github.com/QubesOS/updates-status/issues/6141) (r4.3)
  **app-linux-input-proxy** [v1.0.44](https://github.com/QubesOS/updates-status/issues/6140) (r4.2)
A very interesting and exciting patch from security perspective. It is now possible to allow only certain USB input devices (Keyboards, Mice, Tablets) based on the vendor ID, Product ID or even the USB port it is connected to. This way, you could avoid most [hardware keyboard loggers](https://en.wikipedia.org/wiki/Hardware_keylogger) which usually could not fake the VID/PID (at the time of this post). This is very useful considering how such loggers have become so common these days everywhere and even [open source implementations](https://github.com/Push3AX/USBKeylogger/tree/main) which could be easily implanted inside original devices are readily available. The only issue with this patch is that the documentation for it is not yet available. Tech-savvy users should not have problem to understand it and use it by reading the patch code and the related Github issue.
