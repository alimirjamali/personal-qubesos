### Highlights
- Qubes OS **r4.3.0 stable** is released!

### Details
In addition to the usual minor fixes and patches (full list [here](https://github.com/QubesOS/updates-status/issues?q=is%3Aissue+created%3A2025-12-15..2025-12-21)):

* **meta-packages** [v4.3.7](https://github.com/QubesOS/updates-status/issues/6294) (r4.3)
  **meta-packages** [v4.3.6](https://github.com/QubesOS/updates-status/issues/6292) (r4.3)
  **meta-packages** [v4.3.5](https://github.com/QubesOS/updates-status/issues/6278) (r4.3)
    - Introduction of **devel repo**. In addition to `stable` and `testing` repositories, a new development repository is born. Some background about this repository could be found [here](https://github.com/QubesOS/qubes-issues/issues/10478). If you look at the list of individual packages uploaded to Qubes OS repository in the collapsed list above, you will see many .deb, .rpm, etc. files with `devel` in package name. Unlike stable and testing repositories where usually historic packages are kept, usually the last `devel` package might be kept. More information about this in the epilogue.

* **qubes-template-debian-12-minimal** [4.3.0-202512180242](https://github.com/QubesOS/updates-status/issues/6290) (r4.3)
  **qubes-template-debian-12-xfce** [4.3.0-202512180024](https://github.com/QubesOS/updates-status/issues/6286) (r4.3)
  **qubes-template-debian-12** [4.3.0-202512180024](https://github.com/QubesOS/updates-status/issues/6285) (r4.3)
  **qubes-template-debian-13** [4.3.0-202512180242](https://github.com/QubesOS/updates-status/issues/6289) (r4.3)
  **qubes-template-fedora-42-minimal** [4.3.0-202512180024](https://github.com/QubesOS/updates-status/issues/6282) (r4.3)
  **qubes-template-fedora-42** [4.3.0-202512180242](https://github.com/QubesOS/updates-status/issues/6288) (r4.3)
    A fresh set of Fedora, Debian 12 & Debian 13 templates with the latest updates and patches.

* **qubes-release** [v4.3-1](https://github.com/QubesOS/updates-status/issues/6281) (r4.3)
    Users on r4.3-rc4 will be automatically upgraded to r4.3.0 stable. An installation ISO is build and uploaded. Official announcement is already posted [here](https://github.com/QubesOS/qubes-issues/issues/10478) on forum. Users on r4.2.4 may consider upgrading to the new release. As usual keeping backups is strongly recommended.

* **dist-upgrade** [v4.2.3](https://github.com/QubesOS/updates-status/issues/6279) (r4.3)
  **dist-upgrade** [v4.2.3](https://github.com/QubesOS/updates-status/issues/6280) (r4.2)
    - `qubes-dist-upgrade` tool receives many patches. This is already available in stable repositories. If you are planning in-place upgrade, you should assure you have at least this version.
    - Measures for users who have/had multiple display managers installed.
    - Measures for users who have Whonix is their updatevm or multiple chain of NetVMs.
    - Skipping templates without Qubes qrexec installed (e.g. Windows, BSD variants, vanilla ISO installs...).
    - Skipping already upgraded templates (in case user needs to manually fix something during stage 4).
    - Measures in case of partial stage 3 upgrades.

#### Epilogue
With the birth of development repository, I should reconsider the format and future of this newsletter. Currently I track the testing branch and post changes based on testing packages. While most users would most probably receive such updates days or weeks after that in stable branch. Also new features might not land in stable branch for months (or over a year). The devel branch is similar to Fedora Rawhide or Debian Sid where many things might break. And some changes might be reverted there do not land even in testing branch. But it would be still very exciting to follow.
