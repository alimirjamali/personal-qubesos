### Highlights
- Fixes for preloaded disposables.
- More work on Fedora 43 (template).
- Other general fixes.

### Details
In addition to the usual minor fixes and patches (full list [here](https://github.com/QubesOS/updates-status/issues?q=is%3Aissue+created%3A2025-08-25..2025-08-31)):

* **core-agent-linux** [v4.3.30](https://github.com/QubesOS/updates-status/issues/6008) (r4.3)
. Fixing an issue of building `core-agent-linux` on Fedora 43.
. Chrome web app launchers have a very long `.desktop` filename which was incompatible with `qubes.StartApp` service.  A fix in linux agent and another in core allows using such launchers via hashes. This is mostly useful for open-in-disposable scenarios. Otherwise, it appears that the original issue still remains?

* **core-qrexec** [v4.3.11](https://github.com/QubesOS/updates-status/issues/6007) (r4.3)
. Fixing an issue of building `core-qrexec` on Fedora 43.
. Fixing an issue with mouse not working in `sys-gui-gpu` due to policy issues.

* **core-admin** [v4.3.35](https://github.com/QubesOS/updates-status/issues/6005) (r4.3)
. This release includes many fixed related to preloaded disposables. Some are listed below.
. Issues with backup and restore of disposable templates with preload enabled.
. Fixing an issue with preloaded disposables not starting automatically.
. Some patches related to benchmark of dispvms during integration tests. Avoiding preload feature to conflict with test results.
. Fix for and issue of preloaded disposables with suspend.

* **core-admin-client** [v4.3.21](https://github.com/QubesOS/updates-status/issues/6004) (r4.3)
. Fixing nested Python argument parser groups. This was necessary to support Python 3.14 and Fedora 43

* **notification-proxy** [v1.0.5](https://github.com/QubesOS/updates-status/issues/6002) (r4.3)
. The new Qubes Notification proxy (on GUIVM side) sub-process did not exit on qube shutdown. This prevented proper qube shutdown cleanup and zombie domains on `xl list`. This issue is now fixed.
. P.S.: The linked issue shows a minimal payload (only minor and major protocol version passed from vm-side to GUIVM side). Unfortunately the documentation still lacks the information on detailed specification of the payload structure. This will be necessary for GUI-less or systemd-less qubes such as Mirage firewall which need to bypass `d-bus` and write the notification message directly.

* **vmm-xen** [v4.19.3-2](https://github.com/QubesOS/updates-status/issues/6000) (r4.3)
  **vmm-xen** [v4.17.5-12](https://github.com/QubesOS/updates-status/issues/6001) (r4.2)
. Fixing an S3 suspend regression, found on the Novacustom V54 systems.
. Special `xen-hypervisor-debug` is built to ease debugging and tracing of errors. If installed, it will be available in GRUB2 advanced sub-menu. This is currently available only on r4.3 rc1.

* **desktop-linux-manager** [v4.3.18](https://github.com/QubesOS/updates-status/issues/5999) (r4.3)
A problem found with Qubes OS r4.3 rc1 clipboard notification handling (a race condition) which is now fixed.

* **linux-firmware** [v20250808-1](https://github.com/QubesOS/updates-status/issues/5998) (r4.2)
This is the same Linux firmware package from last week which is now packaged for Qubes OS r4.2. Please see last week newsletter for more information.
